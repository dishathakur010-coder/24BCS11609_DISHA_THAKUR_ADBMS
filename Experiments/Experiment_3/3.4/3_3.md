# LeetCode 183 - Customers Who Never Order

## 📌 Problem Statement

Given two tables:

- **Customers** – Contains customer information.
- **Orders** – Contains customer orders.

Write a SQL query to find the names of customers who **have never placed an order**.

The output column should be named **Customers**.

---

## 🗂️ Table Schema

### Customers

| Column | Type |
|--------|------|
| id | int |
| name | varchar |

- `id` is the primary key.
- Each row contains the customer's ID and name.

### Orders

| Column | Type |
|--------|------|
| id | int |
| customerId | int |

- `id` is the primary key.
- `customerId` is a foreign key referencing `Customers.id`.

---

## 📝 SQL Query

```sql
SELECT Name AS Customers
FROM Customers
WHERE id NOT IN (
    SELECT customerId
    FROM Orders
);
```

---

## 📖 Explanation

- The subquery retrieves all customer IDs that have placed an order.
- `NOT IN` filters out those customers.
- Only customers whose IDs are **not present** in the `Orders` table are returned.
- The output column is renamed to **Customers** using `AS`.

---

## 📥 Example

### Customers Table

| id | name |
|---:|------|
| 1 | Joe |
| 2 | Henry |
| 3 | Sam |
| 4 | Max |

### Orders Table

| id | customerId |
|---:|-----------:|
| 1 | 3 |
| 2 | 1 |

### Output

| Customers |
|-----------|
| Henry |
| Max |

---

## ⏱️ Time Complexity

- **O(n + m)**

where:

- `n` = number of customers
- `m` = number of orders

---

## 💡 Alternative Solution (LEFT JOIN)

```sql
SELECT c.name AS Customers
FROM Customers c
LEFT JOIN Orders o
ON c.id = o.customerId
WHERE o.customerId IS NULL;
```

---

## 🔑 Key Concepts

- `NOT IN` filters rows that do not exist in another query.
- Subqueries can be used inside the `WHERE` clause.
- `AS` renames the output column.
- `LEFT JOIN` with `IS NULL` is another common way to solve this problem.

---

## ✅ Result

The query returns the names of all customers who **have never placed an order**, with the output column named **Customers**.
